# idea-spark · 灵感火花

> 从书、资料、问题、趋势和聊天记录里，挖出有用有趣的小产品机会。

`idea-spark` 是一个 AI agent skill。它不负责判断一个想法能不能赚钱，也不负责设计完整产品；它专注做上游创意工作：

```text
原始资料 / 模糊问题
-> 机会地图
-> Top 10 小产品机会
-> 淘汰弱候选
-> Top 3 候选
-> 交给 business-lens 判断商业逻辑
```

## 装上就能用

打开你正在用的 AI agent，告诉它：

```text
帮我安装这个 skill：https://github.com/ghostyee2023/idea-spark
```

或者用通用 CLI 安装器：

```bash
npx skills add ghostyee2023/idea-spark
```

## 适合什么场景

- 从一本书、一篇文章、一份课程资料里找小产品机会
- 从客户抱怨、聊天记录、会议纪要里提炼产品化方向
- 从一个行业问题或热点趋势里发散小工具、模板、插件、Agent、服务 MVP
- 在还没有明确 idea 时，先生成候选机会池

## 和另外两个 skill 的关系

```text
idea-spark      灵感火花：找机会
business-lens   商机透镜：判断商业逻辑
maker-forge     造物工坊：锻造成 3 小时 MVP
```

没有想法时先用 `idea-spark`。  
想判断有没有商机时用 `business-lens`。  
要做第一版时用 `maker-forge`。

## 能做什么

| 能力 | 说明 |
| --- | --- |
| 入口分流 | 判断输入是原始资料、模糊需求、已有 idea，还是已有候选清单 |
| 机会地图 | 提炼痛点、重复流程、信息差、决策困难和可工具化节点 |
| 多形态发散 | 生成工具、模板、插件、知识库、服务 MVP、诊断产品等候选 |
| 候选审计 | 保留候选、通过项、淘汰项和淘汰原因 |
| 压力测试 | 检查 Top 3 是否真的不同、能不能构建、边界在哪里 |
| source-pack | 把可复用资料编译成机会资产包 |

## 示例提示词

```text
用 idea-spark 分析这篇文章，帮我挖 10 个小产品机会。
```

```text
我有一份课程资料，想看看里面能不能提炼出工具、模板或服务型产品。
```

```text
从这段客户聊天记录里找产品化机会，最后给我 Top 3。
```

## 边界

- 它生成的是候选机会，不是市场验证。
- 它不会声称“资料里出现痛点”就等于有人愿意付费。
- 它不会默认写文件，除非你明确要求保存或落地。
- 它不会暴露私有路径、内部项目名或外部方法来源。
