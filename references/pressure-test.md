# Pressure Test

Use this before recommending which opportunity to build.

## Three Test Types

Every serious shortlist should include:

- should-build: why this candidate deserves evaluation by Maker Forge.
- should-not-build: why a tempting adjacent candidate should be rejected.
- edge-case: what situation would make the candidate weak or not worth building.

## Shortlist Test

For each Top 3 candidate, answer:

```text
would a real user recognize this pain in one sentence?
is the input concrete?
is the output concrete?
does the first version fit one sitting?
what adjacent idea could be confused with it?
what would make this idea not worth building?
```

## Confusion Test

The Top 3 must be meaningfully different. If two candidates serve the same user, same scene, same input, and same output, merge them or reject the weaker one.

## Buildability Test

Reject or downgrade if the first version requires:

- broad platform integrations
- proprietary data the user does not have
- high-stakes accuracy
- institutional trust
- a marketplace before one user gets value
