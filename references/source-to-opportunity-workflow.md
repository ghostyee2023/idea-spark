# Source To Opportunity Workflow

Use this workflow to turn source material into product opportunities.

## 1. Source Map

Identify:

- topic
- audience
- repeated scenes
- explicit pains
- implicit pains
- current workarounds
- decisions people struggle with
- information people repeatedly need
- artifacts people create
- handoffs between people or tools

## 2. Productizable Signals

Look for these signal types:

- repeated manual workflow
- high-friction decision
- messy transformation from input to output
- recurring checklist
- scattered knowledge that needs retrieval
- comparison or ranking problem
- personal judgment that can be partially externalized
- template-worthy deliverable
- review, critique, or quality-control step
- lightweight automation around an existing tool

## 3. Opportunity Sentence

Write every candidate as:

```text
For [user], when [scene], help them turn [input/current mess] into [useful output/result], so they can [job-to-be-done].
```

## 4. Product Shape

Choose the smallest plausible shape:

- prompt pack
- template library
- checklist
- diagnostic assistant
- browser extension
- local script
- micro SaaS
- knowledge base
- data table
- content product
- service-assisted MVP

## 5. Non-Product Filter

Downgrade anything that is only:

- a summary
- a quote collection
- generic advice
- broad education
- a vague AI assistant
- an idea with no repeated input
- an idea with no concrete output

The output must improve a real workflow.
