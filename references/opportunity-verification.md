# Opportunity Verification

Use this before handing candidates to Business Lens.

## Three Checks

1. Source support
   The source material must show a real pain, workflow, contradiction, repeated question, or unmet need. If there is only one weak signal, mark it as low confidence.

2. Transferability
   The opportunity should apply beyond the exact source. It must plausibly help a repeatable user group, not only explain the material.

3. Product sharpness
   The first version must have clear input, output, and user action. If it cannot be used in one sitting, narrow it.

## Rejection Reasons

Use one of these labels:

- summary-only
- generic-advice
- no-repeat-workflow
- unclear-user
- unclear-output
- too-broad
- high-trust-burden
- integration-heavy
- no-switching-pressure

## Top 3 Selection

Rank by:

- repeated pain
- current workaround cost
- narrow user
- concrete input/output
- small first version
- reachable first users
- differentiated angle

Prefer a boring repeated workflow over an exciting vague idea.

## Business Lens Handoff Threshold

Hand off to `business-lens` only if the candidate has:

- named user
- named scene
- pain and current workaround
- concrete input
- concrete output
- plausible 3-hour slice
- clear uncertainty to test

Normalize each shortlisted candidate with `references/opportunity-contract.md`. The buyer may remain `unknown`; Business Lens owns payer, budget, value capture, acquisition, and commercial decision.
