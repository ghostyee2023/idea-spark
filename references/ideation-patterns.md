# Ideation Patterns

Use these sanitized method patterns to generate diverse candidates without exposing source origins.

## Multi-Lens Roles

Run the source through these roles:

- Explorer: find surprising pains, edge cases, and underserved users.
- User proxy: ask who would repeatedly need this and what they do today.
- Skeptic: identify why nobody would pay, switch, trust, or repeat usage.
- Product shaper: convert broad possibilities into narrow workflows.
- Builder: reduce each idea to a 3-hour first version.
- Archivist: preserve source signals and rejected ideas.

## Opportunity Lenses

Generate candidates across several shapes:

- Workflow tool: automate or guide a repeated job.
- Decision aid: help users compare, choose, prioritize, or diagnose.
- Transformation engine: convert messy input into a useful artifact.
- Review tool: critique, score, detect risk, or improve quality.
- Micro database: organize scattered cases, examples, or rules.
- Template system: turn expert judgment into reusable formats.
- Learning product: teach a narrow skill with practice and feedback.
- Agent workflow: coordinate multiple steps with checkpoints.
- Plugin: sit inside an existing browser, note app, editor, or publishing workflow.
- Manual concierge: deliver the result by hand before building software.

## Candidate Diversity Rule

For a 10-idea output, include:

- 3 workflow tools
- 2 decision or diagnostic products
- 2 template or content products
- 1 plugin or automation
- 1 data or knowledge-base product
- 1 service-assisted MVP

Adjust the mix if the source material clearly favors another shape.

## Avoid

- ten variants of the same idea
- only SaaS ideas
- only prompt packs
- ideas that require a large platform before the first user gets value
- ideas whose first version cannot be explained in one screen
